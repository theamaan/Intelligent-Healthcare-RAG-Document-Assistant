# Intelligent Healthcare RAG Document Assistant

Local-first, open-source healthcare document assistant using RAG, citations, and offline evaluation.

This project uses open-source components only. No proprietary model APIs are required.

## Stack
- FastAPI backend
- Streamlit frontend
- LangChain orchestration
- FAISS vector store
- sentence-transformers embeddings
- Ollama local LLM runtime
- RAGAS evaluation

## Quickstart (local, no Docker)
1. Create venv and install dependencies:
   - `python -m venv .venv`
   - `.venv\\Scripts\\activate`
   - `pip install -r requirements.txt`
2. Copy `.env.example` to `.env` and adjust if needed.
3. Start Ollama and pull model:
   - `ollama serve`
   - `ollama pull llama3.2:3b`
4. Start backend:
   - `uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000`
5. Start frontend:
   - `streamlit run frontend/app.py`
6. Open Streamlit URL and upload a PDF or DOCX.

## Run Helpers (Windows PowerShell)
- `./scripts/run_backend.ps1`
- `./scripts/run_frontend.ps1`

## Optional Docker
- `docker compose up --build`

## API Endpoints
- `POST /upload`
- `POST /query`
- `GET /history/{doc_id}`
- `GET /health`

## Evaluation
Run offline RAG evaluation:
- `python -m backend.app.evaluation.ragas_eval`

Before running evaluation, update `data/test_sets/sample_qa.json` with real `doc_id` values created from uploads.

## Notes
- Citations are derived from retrieved metadata (source/page/chunk), not generated by the model.
- Uploaded files and vector indices are stored under `data/` and are git-ignored.
